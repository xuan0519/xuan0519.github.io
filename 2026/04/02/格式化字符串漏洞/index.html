格式化字符串漏洞

格式化字符串漏洞产生原因

格式化字符串漏洞产生于C/C++程序中，当程序使用printf、sprintf、fprintf等一系列格式化输出函数时，如果用户提供的输入被直接或间接作为这些函数的第一个参数（即格式化字符串参数），而程序又没有对其进行严格检查和过滤，就会导致该漏洞。

核心原理

格式化输出函数（如printf）的工作方式依赖于格式化字符串中的格式化指示符（Format Specifiers），例如：

  • %s: 输出字符串
  • %d/%i: 输出整数
  • %x/%p: 输出十六进制数/指针地址
  • %n: 特殊且危险，它不输出任何数据，而是将目前为止已输出的字符数写入由对应参数指定的内存地址。

在漏洞场景下，printf函数会根据用户输入的格式化字符串（如%x）去栈上依次寻找并解析其需要的参数。由于没有显式地提供这些参数，它就会从当前函数的栈帧中继续向高地址方向取出数据作为参数来处理。

  • 如果使用%x等，它会泄露栈上的数据。
  • 如果使用%s，它会尝试将栈上的某个值解释为一个地址，并打印该地址指向的内存内容，可能导致程序崩溃或内存信息泄露。
  • 如果使用**%n，它会尝试将一个栈上的值解释为一个内存地址，并将已打印的字符数写入到该地址中，这是实现任意地址写入**的关键。

格式化字符串漏洞利用手法

利用格式化字符串漏洞可以实现多种攻击目的，主要包括信息泄露和任意地址写入。

1. 内存信息泄露（读取）

通过使用格式化指示符，可以查看栈上或内存中的数据。

格式化指示符 目的 示例 效果
%x 泄露栈上数据 AAAA%x.%x.%x.%x 打印栈上的连续4个字（4字节或8字节），AAAA作为定位标记。
%p 泄露指针地址 %p.%p 效果类似%x，但通常会带有0x前缀，更常用于泄露内存地址。
%s 泄露指定地址的字符串 %s 如果栈上某个值恰好是一个有效地址，printf会尝试打印该地址处的字符串，可用于泄露敏感字符串。

利用技巧： 使用位置参数（Positional Arguments）或多次使用格式化符来精确控制要泄露的栈偏移量。例如：在某些C库实现中，可以使用 $n 语法，如 %6$x 表示获取栈上第6个参数。

2. 任意地址写入（覆盖）

这是最危险的利用方式，通过**%n**指示符实现。%n可以将已打印的字符数写入到它对应的参数所指向的内存地址。

① 基础原理：

  1. 确定写入地址： 确定想要修改的内存地址，例如一个函数的**全局偏移表（GOT）**条目、一个函数的返回地址等。
  2. 确定写入值： 确定想要写入的新值（通常是shellcode的地址或另一个函数的地址）。
  3. 构造Payload：
    • 将要写入的地址放置在格式化字符串的开头，作为%n的参数。
    • 使用%[width]x或%[width]d来控制已打印的字符数，使其等于想要写入的值。
    • 使用**%n**来触发写入操作。

② 实际写入步骤（以写入2字节/4字节为例）：

因为直接写入大数值（如一个完整的32位/64位地址）需要打印大量的字符，效率低且容易失败。实际中常采用**多次写入（Two/Four-Byte Writes）**的方式：

  1. 确定目标地址（ Target Address）：比如 addr。
  2. 确定要写入的值（ Target Value）：比如 value。
  3. 分拆写入： 将 value 拆分成多个字节块（如两个16位或四个8位）进行写入。
    • 写入 value 的低 16 位到 addr。
    • 写入 value 的高 16 位到 addr + 2 (针对32位系统)。

Payload 结构示例 (32位系统，写入2个16位值)：

  • addr_1 和 addr_2：是要写入的两个目标地址，用于作为后续 %n 的参数。
  • low_val：要写入的第一个16位值。
  • \% \text{low_val} c：通过字符输出使已打印字符数达到 low_val。
  • \% n：将当前字符数 low_val 写入到 addr_1。
  • \% (\text{high_val} - \text{low_val}) c：继续打印字符，使总字符数达到 high_val。
  • \% n：将当前字符数 high_val 写入到 addr_2。

3. 利用总结

通过上述的信息泄露和任意地址写入能力，攻击者通常可以：

  • 修改GOT表项： 将某个库函数（如puts、printf）的GOT条目地址修改为shellcode地址或system()函数的地址，从而劫持程序流程。
  • 覆盖返回地址： 修改栈上的函数返回地址，使其指向攻击者预先准备好的代码（如shellcode）。
